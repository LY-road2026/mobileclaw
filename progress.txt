========================================
 MobileClaw 项目进度日志
 基于长时运行 Agent 方法论
========================================

项目概述:
  iOS/Android 多模态语音+视觉 OpenClaw 移动端客户端。
  语音唤醒 → 相机+麦克风同时激活 → 实时画面+语音发给 openclaw → AI 回复 + TTS 播放。
  目标用户：袋鼠帝（公众号写作者），日常需要高频与 AI Agent 交互。

技术栈:
  React Native + Expo Dev Client (Bare Workflow) + TypeScript
  Zustand 状态管理 / react-native-vision-camera v4 / React Navigation v6
  ASR: 豆包流式 API (主) / TTS: Edge TTS + openclaw tts.convert (双路径)
  后端协议: OpenClaw Gateway WebSocket (JSON req/res/event, protocol v3)

配色/设计方向:
  暗色主题 (#0a0a0f 背景)，工具优先极简 UI
  相机预览占屏幕 ~55%，聊天记录在下方
  录制状态红色脉冲边框，TTS 播放紫色波形动画

关键发现:
  - openclaw 已有完整 WS 协议（req/res/event JSON 帧），不需要自造协议
  - openclaw 相机是按需 snap/clip（非连续流），需扩展 video_frame 事件实现实时流
  - npm 缓存有 root 权限问题（.npm/_cacache 下有 root 所有权文件），pnpm 可正常使用
  - VisionCamera Frame Processor 必须在独立线程调用 frame.close()，否则秒崩
  - 半双工模式（ASR/TTS 互斥）比全双工简单可靠，Phase 1 够用
  - iOS Siri Shortcut 通过 URL scheme（mobileclaw://activate）比 App Intents 简单 80%
